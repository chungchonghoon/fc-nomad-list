# Nomad List - 프로젝트 학습 및 패턴 (Learnings)

## 개발 패턴
### 데이터 관리
- **데이터 분리 원칙**: 페이지 컴포넌트(`app/page.tsx`) 내에 정적 데이터를 하드코딩하지 않고, `lib/data.ts`와 같은 전용 파일로 분리하여 유지보수성을 높임.
- **타입 정의 위치**: 데이터와 밀접한 관련이 있는 타입(`City`, `Continent` 등)은 데이터 파일(`lib/data.ts`)에 함께 정의하여 `export` 하는 구조를 채택.

### 로컬라이제이션 (Localization)
- **한국어 우선**: 모든 사용자 대면 텍스트(도시명, 국가명, UI 레이블)는 한국어를 기본으로 작성함.
- **명확한 표기**: "속도" 대신 "인터넷 속도", "AQI" 대신 "공기질 지수(AQI)"와 같이 약어보다는 사용자가 이해하기 쉬운 명확한 표현을 선호함.

## UI/UX
- **정보의 명확성**: 수치 데이터(AQI, 속도 등) 표기 시 단위나 설명을 함께 제공하여 모호함을 줄임.
